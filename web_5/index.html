Azure Front Door

Es la red de entrega de contenido (CDN) avanzada en la nube de Microsoft, diseñada para proporcionar acceso rápido, confiable y seguro al contenido web estático y dinámico de aplicaciones a nivel mundial.

Información general

INFORMACIÓN GENERAL

Una CDN, es una red superpuesta de computadoras (red virtual que se construye encima de otra red física) que contienen copias de datos, colocados en varios puntos de una red.

1. El cliente se conecta de forma segura a Azure Front Door.

2. Front Door revisa la solicitud con su WAF (Web Application Firewall) y la bloquea si es peligrosa.

3. Si Front Door ya tiene la respuesta en caché, la envía de inmediato.

4. Si no está en caché, Front Door pide la información al servidor de origen usando la red privada de Microsoft.

5. El servidor de origen envía la respuesta a Front Door.

6. Front Door guarda esa respuesta en caché para futuros clientes.

7. Front Door devuelve la respuesta al cliente.

8. El Firewall del servidor evita accesos directos desde internet.

Servidor proxy

En una red informática, es un servidor —programa o dispositivo—, que hace de intermediario en las peticiones de recursos que realiza un cliente (A) a otro servidor (C).

Más información

Servidor proxy inverso

Es un tipo de servidor proxy que recupera recursos en nombre de un cliente externo, desde uno o más servidores internos. Estos recursos se devuelven al cliente como si se originaran en el propio servidor Web.

SEGURIDAD

Seguridad y conformidad integradas

Actúa como primera línea de defensa para las aplicaciones.

Protege direcciones IP y topología interna, y crea una barrera física contra ataques directos.

Web Application Firewall (WAF) integrado; inspecciona cada solicitud en busca de amenazas (inyecciones SQL o cross-site scripting).

Conexión nativa con Azure Active Directory para identidad, Azure Monitor para detección y Azure Sentinel para respuesta a incidentes.

BALANCEO DE CARGA

Distribución del tráfico para evitar sobrecargas

PERMITE

  • Sistemas más rápidos.
  • Evitar caídas por exceso de usuarios.
  • Mejorar la disponibilidad y estabilidad de los servicios.

APORTA

  • Menor latencia.
  • Estabilidad.
  • Failover.
  • Mejor rendimiento.

TIPOS DE BALANCEO

  • Azure Load Balancer: Distribuye tráfico TCP/UDP (alta disponibilidad).
  • Azure Application Gateway: Balanceo nivel 7 con WAF y enrutamiento por URL.
  • Traffic Manager: Balanceo global basado en DNS.
  • Azure Front Door: Aceleración global + CDN + seguridad.

RENDIMIENTO

Aceleración del tráfico global con arquitectura distribuida

1. Optimiza el flujo de datos entre usuarios y aplicaciones, reduciendo la latencia a nivel mundial.

2. Emplea la infraestructura Edge de Microsoft para servir de forma local, optimizando la experiencia de navegación.

3. Sistema de caché distribuido que almacena contenido estático y dinámico, disminuyendo el tiempo de respuesta y evitando viajes innecesarios al servidor de origen.

4. Balanceo de carga inteligente que dirige el tráfico al servidor con mejor disponibilidad, garantizando rendimiento estable incluso bajo alta demanda.
